# Deepfake 检测平台前端

一个简洁、美观、易用的 Deepfake 检测平台前端应用，与后端 API 完美集成，提供直观的用户界面进行深度伪造检测。

## 🚀 功能特性

### 核心功能
- **🔍 智能检测** - 支持图片和视频文件的深度伪造检测
- **📁 拖拽上传** - 简单直观的文件上传体验
- **🧠 多模型支持** - 支持 VGG、LRCN、Swin Transformer、ViT、ResNet 等多种检测模型
- **📊 实时结果** - 即时显示检测结果和置信度
- **📈 数据统计** - 详细的检测统计和趋势分析

### 管理功能
- **📜 历史记录** - 完整的检测历史查看和管理
- **🔧 模型管理** - 查看和管理可用的检测模型
- **📉 统计分析** - 可视化的检测数据和性能指标
- **🗑️ 记录管理** - 支持删除历史检测记录
- **✅ 训练结果展示** - 展示训练指标与模型文件路径，是否保留模型由人工决定
- **🧾 人工模型决策** - 对已完成训练任务可执行“保留模型/删除模型”
- **🗂️ 数据集下拉选择** - 训练表单直接选择已登记数据集，不再依赖手工反复输入路径
- **📂 双通道数据集导入** - 同时支持登记本地路径和浏览器选择整个文件夹上传
- **🔄 训练任务自动刷新** - 训练页每 1 秒自动轮询任务列表和状态
- **🎯 保留模型可直接检测** - 已保留模型会进入检测模型列表，可直接用于推理
- **🪜 训练阶段可视化** - 训练卡片会展示当前阶段、错误原因和更清晰的完成状态

### 用户体验
- **📱 响应式设计** - 完美适配桌面和移动设备
- **🎨 现代界面** - 简洁美观的用户界面设计
- **⚡ 快速响应** - 优化的性能和流畅的交互

## 📖 使用指南

### 1. 文件检测
1. 在检测页面，点击上传区域或拖拽文件
2. 选择检测模型（默认 VGG）
3. 点击"开始检测"按钮
4. 等待检测完成，查看结果

### 2. 查看历史
1. 点击导航栏的"历史"
2. 使用筛选器过滤记录
3. 点击删除按钮移除不需要的记录

### 3. 模型管理
1. 点击导航栏的"模型"
2. 查看可用模型的状态和统计
3. 了解各模型的性能指标

### 4. 统计分析
1. 点击导航栏的"统计"
2. 查看总体检测数据
3. 分析检测趋势图表

### 5. 模型训练
1. 进入"模型训练"页面，从下拉列表选择已登记数据集
2. 如果列表中没有数据集，点击“新增数据集”
3. 选择其中一种方式导入数据集：
   - 登记本地路径：适合后端可直接访问的大型数据集
   - 选择文件夹上传：适合小中型数据集或浏览器直接上传
4. 提交训练任务并查看自动刷新的训练进度
5. 训练卡片会显示当前处于训练、验证、保存最佳模型或完成等阶段
6. 训练完成后查看模型文件路径
7. 点击“保留模型”或“删除模型”完成人工决策
8. 保留后的模型会出现在检测模型下拉列表中，可直接用于检测

### 6. 数据集导入建议
1. 如果数据集很大，优先使用“登记本地路径”
2. 推荐目录结构：

```text
dataset/
  fake/
  real/
```

3. 也支持显式拆分目录：

```text
dataset/
  train/
    fake/
    real/
  val/
    fake/
    real/
```

4. 浏览器选择文件夹时，前端上传的是文件内容而不是真实绝对路径，这是浏览器安全限制
5. 因此超大数据集更适合先放到后端可访问目录，再在前端登记路径

## 🔍 API 集成

### 检测接口
```javascript
POST /api/v1/detection/detect
Content-Type: multipart/form-data

参数:
- file: 上传的文件
- model_type: 内置模型类型
- model_id: 已保留模型的 ID（优先于 `model_type`）
```

### 历史记录接口
```javascript
GET /api/v1/detection/history
参数:
- skip: 跳过记录数
- limit: 限制记录数
- prediction: 结果筛选
- model_type: 模型筛选
```

### 统计接口
```javascript
GET /api/v1/detection/statistics
返回: 检测统计数据
```

### 训练接口
```javascript
POST /api/v1/training/jobs
Content-Type: application/json

参数:
- name: 任务名称
- model_type: 模型类型
- dataset_path: 数据集路径
- parameters: 训练参数（epochs/batch_size/learning_rate 等）
```

训练任务响应中包含 `results.model_path`，用于人工判断是否保留模型文件。

训练任务响应还会包含 `current_epoch`、`total_epochs`、`progress_message`，前端用这些字段展示训练阶段。UI 中只有真正完成的任务才会显示 `100%`。

```javascript
POST /api/v1/training/jobs/{id}/model/retain
DELETE /api/v1/training/jobs/{id}/model
```

调用保留接口后，前端会自动刷新检测模型列表，保留模型可直接在检测页面中按 `model_id` 使用。

### 数据集接口
```javascript
GET /api/v1/datasets/
POST /api/v1/datasets/
POST /api/v1/datasets/upload-folder
```

- `GET /api/v1/datasets/`：加载训练页数据集下拉列表
- `POST /api/v1/datasets/`：登记后端可访问的本地/服务器路径
- `POST /api/v1/datasets/upload-folder`：上传浏览器所选整个文件夹，保留 `fake/real` 相对目录结构

## 🐛 故障排除

### 常见问题

**Q: 无法连接到后端 API**
A: 检查后端服务是否在 `http://localhost:8000` 运行

**Q: 文件上传失败**
A: 确认文件格式和大小符合要求

**Q: 数据集很大时该用哪种方式导入？**
A: 优先使用“登记本地路径”，避免浏览器整文件夹上传带来的重复传输、超时和额外磁盘占用

**Q: 为什么手动选择文件夹后拿不到本地绝对路径？**
A: 这是浏览器安全限制，因此前端只能上传文件内容；如果需要真实路径，请使用“登记本地路径”

**Q: 训练任务列表会自动更新吗？**
A: 会，训练页会每 1 秒自动刷新一次，同时仍保留手动刷新按钮

**Q: 为什么进度接近 100% 但任务还没结束？**
A: 训练批次跑完后，系统还可能处于验证或保存最佳模型阶段；只有状态真正变为 `completed` 时，前端才会显示 `100%`

**Q: 检测结果不显示**
A: 检查浏览器控制台的错误信息

**Q: 页面样式异常**
A: 清除浏览器缓存或检查网络连接
